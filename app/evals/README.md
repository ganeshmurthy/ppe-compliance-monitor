# LLM Chat Evals

Evaluates the `/api/chat` endpoint by sending predefined questions and detection contexts, then scoring responses against golden answers using DeepEval's GEval (Correctness) metric with a VLLM-backed judge model.

Supports multiple app configs (PPE, bird, etc.) via the `EVAL_DATASET` env var.

## File layout

```
app/evals/
  db_seed_data.sql              # Full DB snapshot (all app_configs), generated by init-eval-db
  datasets/
    ppe/
      eval_dataset.json         # PPE-specific questions + golden answers
    bird/
      eval_dataset.json         # Bird-specific questions + golden answers
```

- `db_seed_data.sql` is shared across all datasets (one snapshot of the 4 core tables).
- Each `datasets/<name>/` contains a manually curated `eval_dataset.json`.
- `EVAL_DATASET` selects which questions to run (default: `ppe`).

## Prerequisites

The full local stack must be running before running evals:

```bash
make local-build-up
```

OpenAI env vars (`OPENAI_API_TOKEN`, `OPENAI_API_ENDPOINT`, `OPENAI_MODEL`) must be set in `.env`.

## Snapshot the database

Before running evals for the first time (or after the live data changes significantly), snapshot the running database:

```bash
make init-eval-db
```

This dumps `app_config`, `detection_classes`, `detection_tracks`, and `detection_observations` into `app/evals/db_seed_data.sql`.

## Run evals

```bash
# Default (PPE)
make eval

# Specific dataset
EVAL_DATASET=bird make eval
```

The eval script automatically:

1. Saves the live database to a volume-backed snapshot file (`/snapshots/live_backup.sql`)
2. Loads the eval seed data from `db_seed_data.sql`
3. Runs the eval questions from `datasets/<EVAL_DATASET>/eval_dataset.json`
4. Restores the original live data from the snapshot (in a `finally` block, even on error)

## Adding a new dataset

1. Create a directory under `datasets/` with your dataset name (e.g. `datasets/traffic/`).
2. Add an `eval_dataset.json` with entries in this format:

```json
[
  {
    "id": "unique_test_id",
    "question": "How many cars are on screen?",
    "description": "Cars: 3 Buses: 1 Summary: 4 vehicles detected.",
    "golden_answer": "There are 4 vehicles on screen: 3 cars and 1 bus."
  }
]
```

3. Make sure `db_seed_data.sql` contains relevant data for this config (run `make init-eval-db` with the app running the appropriate config).
4. Run: `EVAL_DATASET=traffic make eval`
